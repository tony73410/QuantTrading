# Coding Standards

通用规则保持语言无关。已获批的 `market_history` 模块使用 Python 3.11–3.14；该选择不自动约束未来无关模块。

- 正式代码只进入 `src/`，以实际需求确定模块边界。
- 名称表达业务含义；函数/类型保持单一、可说明的职责。
- 公共接口、输入、输出、错误和副作用应显式定义并记录。
- 优先明确的数据结构，避免魔法值、隐式全局状态和字段不明的通用字典。
- 错误不得静默吞掉；失败方式应可测试且不泄露敏感信息。
- 配置与业务逻辑分离，密钥只从安全环境或密钥管理方式加载。
- 避免重复实现，但不得为“更优雅”进行未批准的大范围重构。
- 注释解释不明显的原因、约束和安全边界，不复述代码表面行为。
- 不引入第三方依赖、生成代码或工具链，除非先说明并获得批准。

## Python market-history rules

- 公共模型使用 dataclass、Enum 和 Protocol 明确字段与边界；公开函数使用类型标注。
- 时间在领域层和存储层使用带时区 UTC；请求采用 `[start, end)`。
- 市场价格使用 `Decimal`，不得为缺失 VWAP 伪造 0。
- 外部异常转换为模块异常，GUI 只显示安全的通俗信息。
- 当前未引入独立 formatter、linter 或静态类型检查器；不得把 `compileall` 说成完整类型检查。
